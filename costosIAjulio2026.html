El caché lo cambia todo

El context caching es automático en la API de DeepSeek: si tu app reenvía el mismo system prompt o los mismos documentos de contexto, ese fragmento repetido se cobra a $0.003/M en Flash — un descuento de ~98% sin configurar nada.

Regla práctica: mantén las instrucciones estáticas al principio del prompt, porque el caché funciona por prefijo.

Gratis para empezar

  • DeepSeek: el chat web (chat.deepseek.com) es gratuito, y la API regala 5 millones de tokens a cuentas nuevas (~$8–10 de valor, sin tarjeta, válidos ~30 días).
  • OpenAI: niveles gratuitos en ChatGPT y modelos mini económicos en API.
  • OpenRouter: agrega modelos con etiqueta :free de varios proveedores en un solo endpoint.
  • Google AI Studio: cuota gratuita generosa de Gemini para desarrollo.

Estrategias que multiplican el ahorro

  • Model routing: consulta simple → Flash (barato); consulta compleja → Pro (caro solo cuando hace falta). Ahorro típico: 60–80%.
  • Formato de salida: el output es 2–5x más caro. Limita max_tokens, pide JSON o formatos estructurados y evita el texto de relleno ("Por supuesto, aquí tienes...").
  • Batch processing: para tareas no urgentes (logs, resúmenes masivos). Hasta 50% de descuento.
  • Horario off-peak: DeepSeek ha aplicado descuentos de 50–75% entre 16:30 y 00:30 UTC. Documentado históricamente para V3/R1; verifica si aplica a V4 antes de contar con él.
  • RAG en vez de contexto gigante: recupera solo los fragmentos relevantes; no pegues documentos enteros en cada prompt.
  • Monitorea: OpenRouter y LiteLLM muestran gasto por modelo y permiten cambiar de proveedor sin reescribir código.
Parte 2

LLM local — costo cero por token

La ejecución local es tu red de seguridad para privacidad y experimentación: pagas solo la electricidad, funciona sin internet y expone una API compatible con OpenAI. La clave son los modelos cuantizados (GGUF), versiones comprimidas que usan mucha menos memoria perdiendo muy poca calidad.

Hardware: qué corre dónde (solo CPU)
EquipoModelosVelocidadComando para probar
8 GB RAM (i3 / Ryzen 3) 1–4B: Llama 3.2 3B, Qwen 2.5 3B, Gemma 2 2B, Phi-3 mini (3.8B) 5–15 palabras/seg ollama run qwen2.5:3b
16 GB RAM (i5 / Ryzen 5) 7–8B: Llama 3.1 8B, Qwen 2.5 Coder 7B, Mistral 7B 3–8 palabras/seg ollama run qwen2.5-coder:7b

La RAM importa más que la generación del procesador: un Ryzen 5 moderno rinde mejor que un i3 antiguo por tener más núcleos.

  • Con GPU: 30–100+ palabras/seg, experiencia tipo ChatGPT.
  • NVIDIA funciona directo (CUDA); AMD requiere algo más de configuración (ROCm).
  • Mejor calidad/precio: RTX 3060 12GB usada + 16 GB de RAM = modelos de 14B con fluidez.

Instalación en 3 pasos (sin interfaz gráfica: Ollama)

# 1. Instalar (Linux/Mac; en Windows descarga el .exe de ollama.com)
curl -fsSL https://ollama.com/install.sh | sh

# 2. Descargar y correr un modelo
ollama run qwen2.5:3b

# 3. Verificar la API local (compatible con OpenAI)
curl http://localhost:11434/v1/models
ollama list          # ver modelos instalados
ollama ps            # ver qué modelo está en memoria
ollama rm llama3.2   # borrar un modelo

Con interfaz gráfica

  • LM Studio (lmstudio.ai): descarga modelos con un clic, chat tipo ChatGPT, también expone API local. La opción más amigable.
  • Open WebUI: interfaz web tipo ChatGPT montada sobre Ollama, con historial y multiusuario.
  • Jan y GPT4All: alternativas open source ligeras.

Conectar Cline a tu modelo local (programar con IA gratis)

  • En VS Code, abre Cline → Settings
  • API Provider: selecciona Ollama
  • Base URL: http://localhost:11434
  • Model: qwen2.5-coder:7b (el equilibrio perfecto entre velocidad y capacidad para programar)
  • Listo: autocompletado y agente de código sin pagar tokens.

Usar tu modelo local desde Python (mismo SDK de OpenAI)

from openai import OpenAI

client = OpenAI(
    base_url="http://localhost:11434/v1",
    api_key="ollama"  # cualquier valor, no se valida
)

respuesta = client.chat.completions.create(
    model="qwen2.5:3b",
    messages=[{"role": "user", "content": "Explica qué es un token en 2 líneas"}],
    max_tokens=100  # limitar output = buena práctica también en APIs de pago
)
print(respuesta.choices[0].message.content)

El mismo código funciona contra DeepSeek cambiando solo base_url="https://api.deepseek.com" y tu API key: por eso conviene programar siempre con el formato OpenAI.
